# A03 — Test / contract first

```yaml
id: A03
version: 1
status: active
supersedes: null
thesis: Failing tests + API contracts written before implementation
```

## Must

- Write failing tests (and/or OpenAPI/contract fixtures) before production code
- Red → green → refactor visible in commits or notes
- Contracts checked in CI

## Must not

- Implementation commits before the failing test commit (same slice)
- Weakening tests to pass

## Review model

Review tests/contracts for sufficiency before or with code review.

## Team notes

| Size | Compression |
|------|-------------|
| 1 | AI writes tests first; human spot-checks AC coverage |
| 2 | A defines AC; B/AI writes tests; then impl |
| 3+ | QA lane can own contract suite |

## Done when

Tests existed while red; oracle green; metrics recorded.
