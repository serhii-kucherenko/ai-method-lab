## Must

- Spec kit: problem, users, AC, non-goals, constraints, open questions
- Task list generated from spec (not ad-hoc mid-coding)
- Spec updated when scope changes (versioned)

## Must not

- Skipping spec for “speed”
- Implementing out-of-spec items without spec revision

## Review model

Spec review before build; code reviewed against spec AC.

## Team notes

| Size | Compression |
|------|-------------|
| 1 | AI drafts full kit; human trims |
| 2 | A owns spec; B owns task breakdown |
| 3+ | Spec is shared source of truth |

## Done when

Spec kit + derived tasks linked; oracle maps to AC; metrics recorded.
