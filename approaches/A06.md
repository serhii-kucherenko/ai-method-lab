# A06 — Small-slice trunk

```yaml
id: A06
version: 1
status: active
supersedes: null
thesis: Tiny PRs, high cadence, feature flags when needed
```

## Must

- Prefer ≤ ~400 LOC churn per PR (soft cap; justify if larger)
- Multiple PRs per project if needed; each mergeable alone
- Feature flags for incomplete behavior on main

## Must not

- Long-lived feature branches spanning days without merges
- Big-bang PR at the end

## Review model

Fast review; reject oversized PRs unless justified.

## Team notes

| Size | Compression |
|------|-------------|
| 1 | Discipline on slice size |
| 2+ | WIP limits; parallel slices only if non-conflicting |

## Done when

≥ 2 merged PRs for non-trivial projects; oracle green; metrics recorded.
