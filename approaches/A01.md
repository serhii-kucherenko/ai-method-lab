# A01 — Thin PRD → code

```yaml
id: A01
version: 1
status: active
supersedes: null
thesis: Minimal docs; AI implements; human merge gate
```

## Must

- One-paragraph PRD with acceptance criteria (bullet list OK)
- Implement immediately after PRD; no separate ERD doc unless schema is non-obvious
- CI: lint + unit tests before PR
- Human (or designated owner) merges

## Must not

- Multi-page specs before first code
- Skipping acceptance criteria
- Editing oracles/harness during the run

## Review model

Author AI self-check + human merge skim. No dedicated adversarial reviewer.

## Team notes

| Size | Compression |
|------|-------------|
| 1 | Human writes PRD intent; AI codes; human merges |
| 2 | A owns PRD/accept; B owns merge review |
| 3+ | Same; WIP limit 1–2 slices to avoid thrash |

## Done when

Oracle green, PR merged (or merge-ready with CI green), metrics recorded.
