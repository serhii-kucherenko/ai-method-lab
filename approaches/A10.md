## Must

- Authz boundaries stated; secrets never committed
- Migration (or explicit “no migration”) note
- Basic observability: structured logs or health endpoint
- Threat notes for new attack surface (short)

## Must not

- “We’ll add security later”
- Shipping without rollback/migration story when schema changes

## Review model

Checklist gate: security + migration + observability before merge.

## Team notes

| Size | Compression |
|------|-------------|
| 1 | Checklist enforced by CI + human |
| 2+ | Security-minded reviewer on each PR |

## Done when

Checklist artifacts present; oracle green; metrics recorded.
