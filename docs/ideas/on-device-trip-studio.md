# On-Device Trip Studio

**State:** ready_to_build  
**Slug:** `on-device-trip-studio`  
**Paper:** https://arxiv.org/abs/2607.15552v1  
**Authors’ code:** none  

## Buyer
Mobile travel teams who need feasible on-device itineraries that respect hard planning constraints before chasing subjective desirability — not cloud-only LLM trip drafts that break schedules.

## Outcome
A studio to open trips, encode constraints, capture desires, run Plan→Learn→Adapt passes, review adapted plans, and compare against desirability-first baselines that skip feasibility.

## Why this paper
Clear on-device constrained planning software claim (PLA). Distinct from Pack Rules (packing lists). No public authors’ code — Method Lab product inspired by the pattern, not a rebrand of PLA.
