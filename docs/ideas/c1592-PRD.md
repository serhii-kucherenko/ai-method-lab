# c1592 — product requirements (PRD, paper)

**Paper only.** Hours hold. No `projects/c1592/` until flip.

## Problem

Trade teams model a single customs civil-penalty number. **19 U.S.C. § 1592(c)** sets different **statutory maximums** by culpability and by whether duties/taxes/fees were lost — including lesser-of domestic value vs N× loss, and % of dutiable value when there is no duty loss. Flat “2× duties” spreadsheets miss branches.

Counsel/CBP own real cases (Kill A). This is a **statutory-maximum forecast** workflow — not mitigated guideline parity.

## Personas

| Persona | Role | Needs |
|---------|------|-------|
| Trade compliance analyst | analyst | Enter facts; run max forecast; batch |
| Auditor | auditor | Read + goldens + audit CSV; no mutate |
| Admin | admin | Members, webhook secret |

## Unique claim

Culpability × (lesser-of domestic vs 2×/4× duty_loss | 20%/40% dutiable when no loss); fraud = domestic value. Reject flat-2× / dual-approver / ignore-domestic-cap cheats.

## Core stories

1. Catalog violations (org-scoped, paginated)  
2. Edit facts + POST forecast → `penalty_max` + `branch`  
3. Batch independent runs  
4. Audit export  
5. Goldens browser  
6. Honesty page (Kill A + fences)  
7. Settings / webhook  

## Non-goals

Mitigated CBP starting points; automatic PD; counsel replacement; dual-gate FSM.
