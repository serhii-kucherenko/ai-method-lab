# Depth restart (2026-07-21)

## Verdict

The portfolio wave failed its real goal. We have **~90 sustained products** that mostly prove the same thing: A03+A10 can ship a tenancy + status FSM + one numeric gate + dual-role close. That is **throughput evidence**, not **comprehensive product** evidence.

## What went wrong

| Symptom | Root cause |
|---------|------------|
| Noun-swap products (yardshare → toyshare → …) | Throughput optimized; clone falsifier in FRAMEWORKS ignored |
| Captable HYPOTHESIS copied from claimreserve | No idea gate before code |
| “Comprehensive wave” still dual-gate template | Domain rules were isomorphic (ceiling/floor/days + dual signer) |
| FINDINGS / digests shallow | Pass counts substituted for insight |
| Research log idle while products multiplied | Research ≠ experiment rule broken in practice |

## What we keep

- Ladder evidence that **A03 (test-first) + A10 (enterprise gates)** is a workable default
- Multi-aggregate shape checklist (tenancy, ≥2 resources, join workflow, domain rule, webhook, pagination)
- Clone falsifier text (now enforced as a hard gate, not a note)

## What we stop

- Queuing the next dual-gate rename
- Marking “sustain complete” as method success when the product is isomorphic to the prior N
- Building before idea depth verification passes

## New optimize-for

**Fewer products. Slower research and idea generation. Bulletproof, well-tested ideas before code.**

Active work = research + idea verification until a candidate clears `protocols/IDEA_DEPTH.md`. Product build resumes only for **one** verified idea at a time, with depth oracles — not smoke clones.
