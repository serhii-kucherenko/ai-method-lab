# Pack Rules Studio — lessons

## What we built

A travel packing studio where hard safety/luggage/dependency rules stay first-class, and soft preferences personalize *inside* that envelope. Dual score compares rules+prefs plans to prefs-only baselines that break constraints.

## What worked

1. **Distinct IA** — profiles / rules / preferences / checklists / optimize / compare reads as packing work, not a renamed jobs desk.
2. **Dual score as the claim** — A holds rules then personalizes; B maximizes prefs and leaks safety/luggage. Goldens prove the gap.
3. **Buyer-first landing** — “lists that never break hard rules” sells an outcome travel product teams recognize.
4. **Soft simulation honesty** — fence early: not a live airline baggage API; paper is research input, not a rebrand.

## Pitfalls

- Do not invent live carrier APIs for sustain theater.
- Do not slip `/jobs` `/lifecycle` `/scenario` back into nav.
- Dual-impl goldens must stay bitwise-equal across `pack.ts` / `packB.ts`.
