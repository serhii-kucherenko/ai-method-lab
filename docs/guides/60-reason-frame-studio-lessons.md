# Lessons — Reason Frame Studio

## What worked

- **Domain-first IA** — `/rules` `/debates` `/scores` `/flags` `/agents` `/compare` reads as a scientific anti-hallucination workflow, not a noun-swapped desk or Agent Safety fleets/monitors shell.
- **Dual score with an honest gap** — multi-agent game-theoretic quality (A) vs single-agent fluent baseline (B) makes the buyer claim falsifiable in goldens and on `/compare`.
- **Landing sells the scientific outcome** — catch fluent-but-false answers before shipping, not a generic “lab desk.”

## What to avoid next time

- Branding as G-Frame or claiming a live production LLM gateway; keep the honesty fence early and visible.
- Copying Agent Safety `/fleets` `/monitors` `/alerts` or desk `/jobs` `/lifecycle` shells — this product needs rule packs, debates, scores, and flags.
- Skipping live `next build` + `next start` smoke; API/unit green alone is not sustain.

## Patterns to reuse

- Dual-impl scoring (`score.ts` / `scoreB.ts`) with ~30 fixtures.
- In-memory store + bearer/webhook/export for comprehensive API bar.
- Tutor guide + `try.html` before finish email.
