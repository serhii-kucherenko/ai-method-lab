Graph Retrieval Desk

Multi-step extract → consolidate → retrieve beats a single opaque hop.

Organize orgs, projects, and graph retrieval jobs for method experiments — a lab desk, not the authors’ production engine.

Open desk Read honesty

The problem

A single opaque retrieve hop hides where coverage is lost. Teams need to see whether separating extraction from consolidation actually improves results — with lifecycle and audit that stay honest under concurrent edits.

The product

Graph Retrieval Desk is an org workspace for retrieval jobs: create work, enforce lifecycle, compare single-hop vs multi-step scoring, batch sibling transitions, and keep an exportable audit. Inspired by the paper; never a stand-in for the authors’ engine.

Why teams use it

  • See when multi-step coverage beats a single opaque hop.
  • Keep every compare, transition, and reject on an audit trail you can export.
  • Run batch sibling updates without one failure poisoning the rest.
  • Browse dual-implementation goldens so desk math stays checkable.
  • Rotate signed webhooks without leaving the org settings page.

Features

Live capabilities grouped for scanning — not a raw CRUD dump.

Org & jobs

  • Org tenancy and member roles

  • Pagination and search

Workflow

  • Lifecycle draft → queued → running → terminal
  • Illegal transition reject

Compare & evidence

  • Scenario: single-hop vs multi-step
  • Audit log and CSV export
  • Goldens browser (≥25 fixtures)
  • Honesty and Sources pages

Integrations

  • Webhook inbound with HMAC and idempotency
  • Admin webhook secret rotate
  • Rate-limit feedback (429 + Retry-After)
  • Offline try page and tutor guide

Honesty / limits

This is a workflow experiment inspired by the paper. It is not a replacement for the authors’ multi-step graph retrieval engine and does not claim measured GraphRAG accuracy from this desk.
