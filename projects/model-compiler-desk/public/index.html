Model Compiler Desk

Treat the model as layered passes — not one opaque compile unit.

Track org projects and MLIR-inspired compile jobs for method experiments: lifecycle, scenario compare, audit, and goldens — a lab desk, not the authors’ production compiler.

Open desk Read honesty

The problem

When an LLM compile path is treated as one opaque block, teams cannot see where modularity scores rise or fall. They need a clear side-by-side of layered pass hints versus a monolith — with lifecycle and audit that survive concurrent edits.

The product

Model Compiler Desk is an org workspace for compile jobs: create work, enforce lifecycle, compare opaque vs MLIR-layered scoring, batch sibling transitions, and keep an exportable audit. Inspired by the paper; never a stand-in for the authors’ compiler.

Why teams use it

  • See when layered pass hints beat an opaque monolith on modularity.
  • Keep every compare, transition, and reject on an audit trail you can export.
  • Run batch sibling updates without one failure poisoning the rest.
  • Browse dual-implementation goldens so desk math stays checkable.
  • Rotate signed webhooks without leaving the org settings page.

Features

Live capabilities grouped for scanning — not a raw CRUD dump.

Org & jobs

  • Org tenancy and member roles
  • Project catalog
  • Compile job create, list, patch, delete
  • Pagination and search

Workflow

  • Lifecycle draft → queued → running → terminal
  • Illegal transition reject
  • Optimistic version conflict
  • Independent batch transitions

Compare & evidence

  • Scenario: opaque vs MLIR layered
  • Audit log and CSV export
  • Goldens browser (≥25 fixtures)
  • Honesty and Sources pages

Integrations

  • Webhook inbound with HMAC and idempotency
  • Admin webhook secret rotate
  • Rate-limit feedback (429 + Retry-After)
  • Offline try page and tutor guide

Honesty / limits

This is a workflow experiment inspired by the paper. It is not a replacement for the authors’ MLIR compiler and does not claim measured TPU or LLM speedups from this desk.
