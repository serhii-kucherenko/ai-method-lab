Gate each stage — then measure before done

Stage gates plus tiered long-context, bit-width, and kernel measurements — against naive intuition baselines.

Problem

Teams ship inference ports on short benches alone — assume 4-bit is faster, treat hand GEMM as the ceiling, and let 2k stand in for 10k.

Product

Stage Validate Desk runs stage-validated scoring beside short-bench only, assume 4-bit faster, and assume hand GEMM ceiling baselines, with jobs, audit, and dual-impl goldens. Soft simulation — not a Fermi CUDA engine.

Selling points

  • Compare stage-gated plans against short-bench / assume-4-bit / hand-GEMM baselines
  • Keep stage-validate jobs honest with lifecycle, batch siblings, and audit export
  • Browse dual-impl goldens so stage-gate scores stay checkable
  • Org webhooks and roles so rejects and rotates stay visible

Features

  • Org tenancy and member roles
  • Stage-validate job create / list / patch / delete
  • Lifecycle and batch sibling transitions
  • Scenario compare (naive vs stage-gated)
  • Audit log and CSV export
  • Goldens browser (≥25 fixtures)
  • Honesty page and offline try
  • Signed webhooks and admin rotate

How it works

  1. Open the desk and create a stage-validate job under a workload profile
  2. Run scenario compare — naive short-bench vs stage-gated measurements
  3. Advance lifecycle or batch, then export the audit trail

Honesty / limits

Workflow experiment inspired by the paper. Not a replacement for the authors' Fermi CUDA engine. Never brand as MiniCPM, Fermi, or Tesla C2075 as the product name.
