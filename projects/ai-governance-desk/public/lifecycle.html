plan job lifecycle

Jobs move only along legal edges: draft → queued → running → succeeded / failed (or cancel from draft, queued, or running). Illegal jumps are rejected.

  1. Create a job (starts as draft, or queue immediately).
  2. POST …/jobs/:id/transition with { "to": "queued" } then running, then a terminal status.
  3. Each move writes an audit row (actor, from, to, time).
