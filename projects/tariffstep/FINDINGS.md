# Tariffstep findings

## Status

**Sustained** through first working version → list/create → workflow → integrations → scale (31 automated checks).

## Framing

Workflow experiment, not a commercial pitch. Existing billing systems still win in the market.

## Unique claim held

- Energy charge walks usage through ordered rate blocks
- Billing demand uses the larger of this month’s peak and prior peak × ratchet percentage
- Bad schedules and illegal percentages reject; rejected bills cannot post

## Takeaway

The standard product test ladder can sustain a **block + ratchet billing math** product without collapsing into a shallow dual-approval template.
