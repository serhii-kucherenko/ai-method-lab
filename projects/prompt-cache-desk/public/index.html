Prompt Cache Desk

Compress once; keep the cheap cache tier

Score query-agnostic compression with a tier-preserving ratio bound against naive baselines under a two-tier hit-rate cost model.

Open desk Read honesty

Problem

Query-aware compressors rewrite the prefix every call and wipe the cache. Over-compression can also push a shared prefix into the hot tier where hit rates plateau below one.

Product

Prompt Cache Desk is an org desk that runs cache-aware compression against vanilla, cache-only, and query-aware baselines, with jobs, lifecycle, audit, and goldens so the claim stays inspectable.

Selling points

  • Compare cache-aware compression against vanilla, cache-only, and query-aware baselines in one scenario
  • Keep cache jobs honest with lifecycle, batch siblings, and audit export
  • Browse dual-impl goldens so scores stay checkable, not theatrical
  • Org webhooks and roles so rejects and rotates stay visible

Features

  • Org tenancy
  • Member roles
  • Project / workload catalog
  • Cache job CRUD
  • Lifecycle transitions
  • Illegal transition reject
  • Optimistic version conflict
  • Batch sibling transitions
  • Scenario compare
  • Audit log
  • CSV audit export
  • Goldens browser
  • Honesty page
  • Signed inbound webhooks
  • Admin webhook rotate
  • Pagination + search
  • Rate-limit feedback
  • Offline try page

How it works

  1. Open the desk and create a cache job under a workload project
  2. Run scenario compare — query-aware vs cache-aware under two-tier ρ
  3. Advance lifecycle or batch, then export the audit trail

Honesty / limits

Workflow experiment inspired by the paper. Not a replacement for the authors' system. Not a commercial LLM API gateway. Never brand as CAPC, Sonnet, or PayPal.
