The problem

Controllers often budget one stacked percentage for late returns. Real additions walk month by month: when both file and pay apply in the same month, the file slice is reduced; levy-intent can raise the pay rate; caps and a minimum floor can bind. A spreadsheet that ignores that stack understates or overstates cash risk.

The product

Filing Penalty Desk is an org workspace for return timelines: edit unpaid months, run the locked addition forecast, compare a naive stack against the correct walk, batch sibling forecasts, and export an audit trail. Built for method honesty — not assessment, abatement, or advice.

Why teams use it

  • See same-month file reduction instead of stacking full file and pay additions.
  • Model an optional levy-intent pay bump after a chosen index month.
  • Compare naive vs correct scenarios side by side before you brief finance.
  • Keep every forecast and edit on an exportable audit trail.
  • Browse dual-implementation goldens so the month-walk stays checkable.

Honesty / limits

This is a cash-forecast / method honesty desk for controllers and tax leads. It is not an IRS assessment, abatement tool, or CPA replacement. Commercial tax software and the IRS still win in production. Never branded as a statute code.

Sources

Not paper-sourced. This desk encodes a literal-dollar month-walk for late-file and late-pay additions as a Method Lab workflow experiment — no published paper or authors’ code URL.
