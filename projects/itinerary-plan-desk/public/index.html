Itinerary Plan Desk

Preferences rank stops; day windows must still hold

Score feasibility-first day scheduling against a flat naive preference-only baseline — in a method-lab desk, not a commercial trip planner.

Open desk Read honesty

Problem

Preference-only trip assistants stack beloved stops and still break opening hours or the day budget. Teams need a desk that keeps hard feasibility while maximizing soft preferences inside the feasible set.

Product

Itinerary Plan Desk is an org desk that runs feasibility-first itinerary scheduling against a flat naive preference-only baseline, with jobs, lifecycle, audit, and goldens so the claim stays inspectable.

Selling points

  • Compare feasibility-first itinerary planning against naive preference-only scoring in one scenario
  • Keep itinerary jobs honest with lifecycle, batch siblings, and audit export
  • Browse dual-impl goldens so scores stay checkable, not theatrical
  • Org webhooks and roles so rejects and rotates stay visible

Features

  • Org tenancy
  • Member roles
  • Project / trip catalog
  • Itinerary job CRUD
  • Lifecycle transitions
  • Illegal transition reject
  • Optimistic version conflict
  • Batch sibling transitions
  • Scenario compare
  • Audit log
  • CSV audit export
  • Goldens browser
  • Honesty page
  • Signed inbound webhooks
  • Admin webhook rotate
  • Pagination + search
  • Rate-limit feedback
  • Offline try page

How it works

  1. Open the desk and create an itinerary job under a trip project
  2. Run scenario compare — naive preference-only baseline vs plan/learn/adapt
  3. Advance lifecycle or batch, then export the audit trail

Honesty / limits

Workflow experiment inspired by the paper. Not a replacement for the authors' on-device itinerary system. Not a commercial trip planner. Never brand as FlyEnJoy or PLA.
